# 🚀 Plan Buddy 프로젝트 개발 여정 및 기술 요약

이 문서는 초보 개발자의 눈높이에서 **Plan Buddy** 프로젝트의 시작부터 현재까지의 발전 과정을 정리한 가이드입니다.

---

## 📅 1단계: 함께하는 즐거움 (Social & Sharing)
혼자 쓰는 달력을 넘어 친구와 일정을 공유하는 소셜 기능을 구현했습니다.

### 주요 구현 사항
- **친구 요청 및 수락 시스템**: 
  - 요청을 보내면 `PENDING`(대기) 상태가 되고, 상대방이 수락해야 `ACCEPTED`(수락) 상태로 변경되는 로직을 구현했습니다.
- **공개 범위(Visibility) 설정**: 
  - 각 일정마다 `PUBLIC`(전체공개), `FRIENDS_ONLY`(친구공개), `PRIVATE`(나만보기) 옵션을 주어 프라이버시를 보호했습니다.
- **공유 달력 사이드바**:
  - 내 일정뿐만 아니라 수락된 친구들의 목록을 보여주고, 클릭 시 해당 친구의 일정을 불러오는 인터페이스를 구축했습니다.

---

## 👤 2단계: 나만의 공간 (My Page Implementation)
사용자 개인의 정보를 관리하고 개성을 표현할 수 있는 기능을 추가했습니다.

### 주요 구현 사항
- **전용 마이페이지(MyPageView)**: 
  - 별도의 페이지(`/mypage`)를 구축하여 닉네임, 비밀번호, 프로필 사진을 한곳에서 관리하게 했습니다.
- **이미지 업로드 시스템**:
  - `Multipart/Form-Data`를 활용해 사용자가 직접 파일을 서버에 올리고, 서버는 이를 `static/profiles/` 폴더에 저장하도록 했습니다.
- **닉네임 중복 검사**:
  - 사용자가 닉네임을 변경할 때 데이터베이스를 조회하여 이미 존재하는 이름인지 확인하는 안전장치를 마련했습니다.

---

## 🛠 3단계: 시스템 안정화 및 UI 개선
앱의 완성도를 높이고 보안 및 사용성을 강화하는 작업을 수행했습니다.

### 주요 구현 사항
- **ID 기반 인증 시스템 (JWT Refactoring)**:
  - **기존 문제**: 닉네임을 신분증(토큰)으로 썼을 때, 닉네임을 바꾸면 신분증이 무효가 되어 로그아웃되는 현상이 있었습니다.
  - **해결**: 변하지 않는 고유 숫자 `ID`를 신분증의 기준으로 변경하여, 닉네임을 언제든 바꿔도 로그인이 유지되도록 개선했습니다.
- **비밀번호 가시성 토글**:
  - 비밀번호 입력창 옆에 버튼을 추가하여 입력 내용을 잠깐 확인할 수 있게 했습니다.
- **사이드바 프로필 실시간 연동**:
  - 마이페이지에서 바꾼 사진이 메인 화면 사이드바에도 즉시 보이도록 API와 UI를 연결했습니다.

## 📝 주니어를 위한 핵심 기술 노트

### 1. JWT (JSON Web Token)
- **개념**: 사용자가 로그인했다는 사실을 서버가 인정해 주는 "디지털 신분증"입니다.
- **중요성**: 서버는 이 토큰을 보고 "아, 이 사람이 7번 유저구나"라고 판단하여 요청을 처리해 줍니다.

### 2. 정적 파일 서빙 (Static File Serving)
- **개념**: 서버에 저장된 이미지나 파일을 브라우저가 직접 주소(URL)로 열어볼 수 있게 해주는 기능입니다.
- **구현**: FastAPI의 `app.mount("/static", ...)` 명령어를 사용해 구현했습니다.

### 3. 데이터베이스 스키마와 create_all()
- **개념**: DB의 '설계도'입니다. 
- **주의점**: 이미 만들어진 설계도에 새로운 칸(컬럼)을 추가할 때는 기존 테이블을 지우고 다시 만들거나, 전문 도구(Alembic 등)를 써야 함을 배웠습니다.
